üß† What Makes a "Language Skill"?

A language skill isn't a text book. It's a Cheat Sheet for the AI. Claude already knows Kotlin syntax (mostly). What it doesn't know is: 1. Your Version: Are you using Kotlin 1.9 or 2.0? 2. Your Libraries: Ktor or Spring? Arrow or standard lib? 3. Your Idioms: Do you prefer runCatching or try-catch? We need to create a lang-kotlin skill to align Claude with your preferences.

<h1>Kotlin Expert Guidelines</h1>

<h2>Quick Reference</h2>

    

  • <strong>Version</strong>: Kotlin 2.0
    • 

  • <strong>Build</strong>: Gradle Kotlin DSL
    • 

  • <strong>Style</strong>: Ktlint standard
    • 



<h2>Idioms to Follow</h2>

<h3>1. Expressions over Statements</h3>
<strong>YES</strong>:
kotlin val status = if (isValid) "OK" else "FAIL" 
<strong>NO</strong>:
kotlin var status = "" if (isValid) { status = "OK" } else { status = "FAIL" } 

<h3>2. Coroutines</h3>
Always expose suspend functions for I/O.
Use Flow for streams, not RxJava.

<h3>3. Null Safety</h3>
Never use !!. Use ?. or ?: or requireNotNull().
kotlin val name = user?.name ?: "Guest" 

<h3>4. Testing</h3>
Use Kotest style assertions if available, or JUnit 5.
kotlin assertThat(result).isEqualTo(expected)

When you save this file, every time you open a .kt file, MoAI will inject these rules into Claude's brain. You will never have to say "Don't use !! please" again. ---
